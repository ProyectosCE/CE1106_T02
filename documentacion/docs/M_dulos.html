3 Módulos

3.1 game_logic

 (require "game_logic.rkt") package: base
Este módulo contiene la lógica del juego. Se encarga de manejar la lógica de detección de ganadores, empates y los movimientos del bot.

3.2 Funciones del Módulo game_logic

procedure

(winner? grid) → boolean?

  grid : (listof (listof symbol?))
{**Descripción:** Verifica si hay un ganador en el tablero actual representado por grid. Se analizan filas, columnas y diagonales para detectar un posible ganador.

**Entradas:** - grid: Matriz bidimensional de símbolos que representa el tablero del juego.

**Salidas:** - Devuelve #t si hay un ganador. - Devuelve #f en caso contrario.

**Restricciones:** El tablero debe tener al menos 3 filas y 3 columnas para que pueda haber un ganador válido.}

procedure

(draw? grid) → boolean?

  grid : (listof (listof symbol?))
{**Descripción:** Determina si el juego es un empate. Un empate ocurre cuando todas las celdas están ocupadas y no hay un ganador.

**Entradas:** - grid: Matriz bidimensional de símbolos que representa el tablero del juego.

**Salidas:** - Devuelve #t si el tablero está lleno y no hay un ganador. - Devuelve #f en caso contrario.

**Restricciones:** El tablero debe estar completamente lleno y no haber un ganador para que se declare un empate.}

procedure

(greedy-bot grid player) → (list integer? integer?)

  grid : (listof (listof symbol?))
  player : symbol?
{**Descripción:** Calcula el mejor movimiento para el bot utilizando una estrategia ’greedy’ (codiciosa). Intenta ocupar la mejor posición disponible en el tablero para el jugador especificado.

**Entradas:** - grid: Matriz bidimensional de símbolos que representa el tablero del juego. - player: El símbolo ('X o 'O) del jugador que está jugando.

**Salidas:** - Devuelve una lista con las coordenadas (fila, columna) del mejor movimiento seleccionado por el bot.

**Restricciones:** El bot selecciona la primera celda que maximiza sus posibilidades de ganar o evita que el jugador contrario gane.}

procedure

(get-matrix rows cols init) → (listof (listof symbol?))

  rows : integer?
  cols : integer?
  init : symbol?
{**Descripción:** Genera una matriz de tamaño rows x cols inicializada con el valor init, que representa el estado inicial del tablero.

**Entradas:** - rows: Número de filas de la matriz. - cols: Número de columnas de la matriz. - init: Símbolo con el que se inicializan todas las celdas (por ejemplo, '_).

**Salidas:** - Devuelve una matriz bidimensional donde cada celda contiene el valor init.

**Restricciones:** Las dimensiones de la matriz deben ser positivas.}

3.3 Descripción de la Lógica del Juego

3.3.1 Detección de Ganador

La función winner? es responsable de verificar si alguno de los jugadores ha ganado el juego. Se comprueban todas las filas, columnas y diagonales del tablero para ver si alguna contiene el mismo símbolo (X o O) en todas sus celdas.

3.3.2 Detección de Empate

La función draw? determina si el juego ha terminado en empate. Esto sucede cuando todas las celdas del tablero están ocupadas y no hay un ganador.

3.3.3 Movimiento del Bot

La función greedy-bot utiliza una estrategia codiciosa para determinar el mejor movimiento del bot. Analiza el tablero y selecciona la primera celda disponible que maximiza su probabilidad de ganar o bloquea al oponente.

3.3.4 Generación de la Matriz

La función get-matrix genera una matriz que representa el estado inicial del tablero del juego, donde todas las celdas están vacías o contienen un valor inicial específico.
